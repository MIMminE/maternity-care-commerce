# Requirements

## 사용자 기능

- 회원가입/로그인
- 케어 프로필 등록
- 임신 주차별 콘텐츠 조회
- 상품 목록/상세
- 장바구니/주문
- 결제 요청/결제 결과 확인
- 배송 조회
- 상품 리뷰 작성
- 상품 Q&A 작성
- 상담 신청
- 알림 확인

## 관리자 기능

- 회원 검색
- 케어 프로필 조회
- 상품 관리
- 주문/배송 관리
- 상담 처리
- 상품 Q&A 처리
- 리뷰 관리
- 쿠폰 관리
- 마케팅 동의 고객 필터링
- 개인정보 접근 로그 조회
- 운영 통계 조회

## 1차 구현 우선순위

1. 회원/인증과 관리자 권한
2. 케어 프로필과 동의 이력
3. 상품/장바구니/주문
4. 상담/문의와 관리자 처리
5. 감사 로그와 운영 통계

## 구현 상태

- 완료: 회원가입/로그인, JWT 인증, 사용자/관리자 API 권한 분리
- 완료: 사용자 케어 프로필 등록/조회
- 완료: 사용자 동의 이력 조회
- 완료: 관리자 회원 목록/상세 조회
- 완료: 관리자 회원 상세 조회 시 감사 로그 저장
- 완료: 사용자 상품 목록/상세 조회
- 완료: 관리자 상품 등록/수정
- 완료: 사용자 장바구니 담기/조회
- 완료: 장바구니 기반 주문 생성/조회
- 완료: 사용자 상담 신청/조회
- 완료: 관리자 상담 상태 처리와 내부 메모
- 완료: 사용자 상품 문의 등록/조회
- 완료: 관리자 상품 문의 상태 처리
- 완료: 관리자 운영 대시보드 통계
- 완료: 마케팅 동의 고객 필터링
- 완료: 마케팅 대상 조회 시 감사 로그 저장
