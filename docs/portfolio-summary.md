# Portfolio Summary

## 한 줄 소개

임산부/산모 사용자 서비스와 관리자 운영 백오피스를 분리한 모성 케어 커머스 백엔드 포트폴리오입니다.

## 문제 정의

모성 케어 기업은 사용자 앱, 커머스, CS, 운영, 마케팅, 법무가 같은 고객 데이터를 서로 다른 목적으로 사용합니다. 따라서 단순 상품 판매 기능보다 권한, 동의 이력, 감사 로그, 관리자 운영 흐름이 중요합니다.

## 설계 방향

- 사용자와 관리자의 API 경계를 분리했습니다.
- 산모 프로필과 동의 이력을 일반 회원 정보와 분리했습니다.
- 관리자 회원 상세 조회, 산모 프로필 조회, 마케팅 대상 조회에 감사 로그를 남깁니다.
- 상품, 장바구니, 주문은 일반 커머스 흐름을 따르되 주문 시점 가격 스냅샷을 저장합니다.
- 상담과 상품 문의는 CS/운영팀의 상태 처리 흐름을 기준으로 설계했습니다.

## 공고와 연결되는 역량

- Java/Spring Boot 기반 백엔드 개발
- Database 모델링과 SQL/JPA 활용
- REST API 설계
- 사용자 API와 관리자 API 권한 분리
- 현업 부서 요구사항을 기능으로 변환
- 개인정보/민감정보 처리 감각
- 커머스/플랫폼 서비스 구조 이해
- Git 커밋 단위 관리와 테스트 작성

## 현재 구현 범위

- Spring Boot API 서버
- React 사용자 모바일 웹/PWA
- React 관리자 백오피스
- JWT 인증
- 산모 프로필
- 동의 이력
- 상품/장바구니/주문
- 상담/상품 문의
- 관리자 통계
- 마케팅 동의 고객 필터링
- 감사 로그
- 통합 테스트
