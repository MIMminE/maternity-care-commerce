# Portfolio Summary

## 한 줄 소개

고객 데스크톱 웹, iOS SwiftUI 앱, 관리자 백오피스, Spring Boot API 서버를 분리한 케어 커머스 운영 시스템 포트폴리오입니다.

화면 중심 설명 문서: [resume-portfolio.md](resume-portfolio.md)

블로그 게시물: [../blog/article.md](../blog/article.md)

## 문제 정의

케어 커머스 서비스는 고객이 상품을 주문하는 화면만으로 끝나지 않습니다. 운영 조직은 회원 상세, 케어 프로필, 상담 상태, 상품 상태, 마케팅 동의 고객을 함께 확인해야 합니다.

따라서 이 프로젝트는 고객 화면과 관리자 화면을 분리하고, 같은 백엔드 도메인을 서로 다른 권한과 목적에 맞게 사용하도록 설계했습니다.

## 설계 방향

- 고객 데스크톱 웹과 iOS SwiftUI 앱을 별도 실행 단위로 분리했습니다.
- 사용자 API와 관리자 API의 경계를 분리했습니다.
- 케어 프로필과 동의 이력을 일반 회원 정보와 분리했습니다.
- 관리자 회원 상세 조회, 케어 프로필 조회, 마케팅 대상 조회에 감사 로그를 남깁니다.
- 상품, 장바구니, 주문은 일반 커머스 흐름을 따르되 주문 시점 가격 스냅샷을 저장합니다.
- 상담과 상품 문의는 CS/운영팀의 상태 처리 흐름을 기준으로 설계했습니다.

## 공고와 연결되는 역량

- Java/Spring Boot 기반 백엔드 개발
- Database 모델링과 SQL/JPA 활용
- REST API 설계
- 사용자 API와 관리자 API 권한 분리
- 현업 부서 요구사항을 기능으로 변환
- 개인정보/민감정보 처리 감각
- 커머스/플랫폼 서비스 구조 이해
- React/TypeScript 기반 화면 구현
- Git 커밋 단위 관리와 테스트 작성
